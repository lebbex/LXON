Definition

A keybind is a value type which can hold any string. The only difference is its slightly more complex syntax, and the fact it doesn't get parsed into a string.

Its primary use, as the name implies, is to store keybinds. In any development environment that supports keybinds, it will get parsed into one, such as in Unreal Engine 5. However, in languages that don't, such as JavaScript, it will get parsed into an alternative. To continue with the JavaScript example, in that language, it gets parsed into a Symbol. In other languages or development environments that have no support for anything but string, it will get parsed into a custom object, allowing it to be distinguished to regular string.

If the project doesn't require keybind parsing, you can either chose to ignore this feature or use it for something else, such as codes. It is generally useful to have a string alternative that doesn't parse into string.

Syntax

A keybind requires the at symbol@as type declaration. Every following character is treated as the contents of the keybind, until the first instance of either one of these characters:

  • Backtick`
  • Caret^
  • At Symbol@
  • Asterisk*
  • Question Mark?

It is standardized however to use the backtick, and only use the other characters for !! val.bind.chain ! chaining ! other values after it. Like with full strings, keybinds also ends at the end of the line, so trailing backticks aren't strictly required. Keybinds also support trailing whitespace such as tabs or space characters.

As a consequence of its increase in complexity, keybinds require escape sequences for the following characters:

  • Newline\n
  • Carriage Return\r
  • Backslash\\
  • Backtick\`
  • Caret\^
  • At Symbol\@
  • Asterisk\*
  • Question Mark\?

Chaining

Keybinds can be chained easily. However, these chains should only ever be done inside an !! containers.arr ! array !. Here's all the possible chained values:

  • When a caret^is used to terminate a keybind, it will not only mark the end of the current keybind but also the start of a new !! val.text.str ! string !.
  • When a at symbol@is used to terminate a keybind, it will mark the end of the current keybind and the start of a new one.
  • When a asterik*is used to terminate a keybind, it will mark the end of the current keybind and the start of a !! val.enum ! enum !.
  • When a question mark?is used to terminate a keybind, it will mark the end of the current keybind and the start of a !! val.binary ! binary ! value.

Important

Although tab representation is supported, it is standardized to replace them with\t. This allows for better visualization as well as a shortening of the visual length of the keybind. This does however come at the cost of an extra byte stored per tab, so this decision comes down to the developper.

Also, although trailing whitespace is supported, if there's no trailing backtick, it is standardized to replace the last character with a\sif it's a space, or a\tif it's a tab. Otherwise, it's easy to miss the trailing whitespace and be unclear as to whether it was intentionally done or not. For this reason, it is very possible for a friendly parser to trim leading whitespace, assuming it's a typing mistake.
