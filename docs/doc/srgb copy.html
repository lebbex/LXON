Syntax

An sRGB color value is the most common way of representing a color in LXON. It requires only a hashtag#at the start, as a type declaration. All succeeding hexadecimal characters (0-9 / a-f) are used to store the color data.

The first two characters converge into the red channel, where "00" gets read as the full lack of red and "ff" reads as the full presence of red. The next two characters converge into the green channel, following the same rules as the red, and the next two become the blue channel. The final character pair converge into the alpha channel, also known as opacity.

The best way to end a color is using a space, although any other non-hexadecimal character would do the trick as well.

Termination

The terminating symbol of a color isn't skipped and is parsed as the start of a new key or value. It is generally good practice to always end a color value by either a space or a newline. Examples of proper and improper color termination (read comments): 

{
# Color bleeding into key (bad)
# (here the color is acctually #ffffaaa2)
aaa1: #ffffaaa2: "value

# Color terminated by the next key (okay)
rrr1: #2e57rrr2: "value

# Color not bleeding into key (recommended)
aaa3: #ffff aaa4: "value

# Color terminated by the next string
array:[ #0000`string`

Channel Skipping

If a certain channel's hexadecimal pair isn't present, the parsing of the color ceases and the current color returns, defaulting the remaining channels. By default, all color channels are empty, except for the alpha channel which holds 100% opacity.

As consequence, if a color happens to have an empty color channel, writing of that data can be skipped. This can be very useful to save characters, especially for skipping the alpha channel which is usually 100% opaque either way.

Channel skipping examples (the keys are named after the colors): 

{
black: #
red: #ff
orange: #ff70
yellow: #ffff
green: #00ff
