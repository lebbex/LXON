Definition

A boolean is a value with two states, hence binary. It can be either true of false. This true/false combination can be used in many instances to store any binary state, such as:

  • Yes / No
  • On / Off
  • Positive / Negative
  • Enabled / Disabled
  • Online / Offline
  • DarkMode / LightMode
  • Male / Female (No political comments being made)

Syntax

A boolean is a !! val.static ! static value !. This means it doesn't acctually require any data to be stored. It only requires the declaration character, that is either$for true or!for false. Once the parser reads this declaration character, it instantly completes the value.

Visualization

A simple trick to remember the characters for booleans is to think of money.

The value true is generally regarded as the presence of something, such as, litterally, the presence of a reality. "online = true" means the presence of the online status. Even "offline = true" means the presence of the offline status. No matter how you look at it, true means the presence of something.

Hence, when thinking of money, true is the presence of money. And what character best symbolizes that? The dollar sign$.

The value false, by consequence, is generally regarded as the lack of something, the lack of truth in a statement. The lack of a status. "online = false" means the absence of the online status, and the same thing applies to "offline = false".

Hence, when thinking of money, false is the absence of money. And when lacking money, you yell. You scream. You feel this terrible feeling, that best gets incapsulated by the exclamation symbol!.

When worded out, it seems complex, but when you think about it, it becomes really obvious.

Usage

  • If the value has only two states, use a boolean.
  • If the value has more then two states (Low, Medium, High), use !! val.enum ! enums ! instead.
  • If the value has two main states and alternative states, you can use different value types and select a value based off the value type. This however requires a bit more code on the interpreter side of things.
