!! other.comments ! Comments !

One thing you'll see a bit in this documentation is comments inside code blocks. You should always read these comments as they sometimes distinguish what's valid syntax and what isn't.

Comments are officially supported, but should rarely be used, and especially not in published projects.

To write a comment, include# as the first character of a newline. The parser will then ignore that entire line.
