!! other.comments ! Comments !

One thing you'll see a bit in this documentation is comments inside code blocks. You should always read these comments as they sometimes distinguish what's valid syntax and what isn't.

Comments are officially supported, but should rarely be used, and especially not in published projects. They can be thought of as a novelty item.

To write a comment, include# as the first character of a newline. The parser will then skip that entire line.

!! other.escape ! Escape Sequences !

An escape sequence is a way to represent a character in an environment where, in order to follow the syntax rules, the character itself cannot be used.

To escape a character, prepend it with\. The parser will ignore the backslash and read only the character following it. If it's a code character, it will replace the pair with the represented character (\n -> newline, \s -> space, etc.), otherwise it will read the second character litterally (\" -> ").
