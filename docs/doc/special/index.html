Definition

A special value is a static value that does not fit into any other value type. It is one of the few value types that aren't also a !! Keys # typed ! typed key ! type, for obvious reasons.

Numbers cannot contain anything other then the number, and they end at the first non-numerical character found. It is recommended to terminate numbers using spaces or newlines in cases where the LXON might be manually edited later.

Regular Numbers

To write a number, you just write the number as is, with a negative sign-in the case of a negative number, or a optional plus sign+in the case of a positive number. 

{
pos: 500
neg: -50
dec: +3.1416

Decimal Numbers

If a decimal number begins with a zero (it's only decimals), you can skip writing the zero and write the number with a period before. Obviously, only one period is allowed in a number; a second period terminates the value and returns it. 

{
valid: .592
extra: -.5
wrong: .623.59
# This last value wouldn't parse correctly

Scientific Notation

Scientific notation is also supported for numbers, and is simply written by appendingeright after the number, as well as a optional sign and the exponent number. 

{
num1: 1.3e5
num2: -2.6e-3
num3: .5e+10

Static Numbers

There are three static values for numbers, each of which self-declared and requiring no type definition:

  • NaN+ - *Must be followed by a non-numerical character
  • Positive (+) Infinity% +%
  • Negative (-) Infinity-%
{
not a number: +
pos infinity: %
neg infinity: -%
