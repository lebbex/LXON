Definition

A date is a value type which can hold any date/time combination value. For the date part of the date, it can store years, including negative ones, or unix timestamps. If year is chosen, it can store as well month/day of the month pairs or week/day of the week pairs, although the latter is much less used.

For the time part of the date, you can precise an hour, minute and second, as well as fractions of seconds going up to millisecond precision. You can also chose to skip precising a date, which results in a date of January 1st, year 0. It is reccomended to recognize that date as the lack of a date, although it is to the developper's discretion how they chose to interpret it.

Year / Unix Timestamp

If the first number is over 11 digits long (12+), it assumes it's looking at a millisecond timestamp and returns the date as a Unix Timestamp. If the first number is over 7 digits long (8+), it assumes it's looking at a seconds timestamp and returns the date as such.

Otherwise, it reads the first number as a year. However, if the year is less then 3 digits long (1 or 2), it assumes a 21st century abbreviation and adds 2000. So, in the rare case you may want to represent, for instance, the year 50, you would have to type it out as 050 otherwise it would read 50 as 2050.

If it had read the number as a year, it then moves onto looking for the week or month, and if that fails, it looks for the time instead.

Here's an example of year and unix timestamp usage. It is the same example as below, representing January 1st, 2026 (UTC): 

{
unix milsec: &1767225600000
unix second: &1767225600
simple year: &2026
21cent year: &26

Week

If, following the year, is the characterW(with or without a dash in front), then it will read the number as the week of the year. If, after that number, is a-, then it will read the number following that as the day of the week, starting from 1 being Monday.

Here's an example of a date made up of a year, a week and day of the week. It is the same example as above and below, representing Januay 1st 2026 (1st week of the year on a thursday): 

{
date: &2026-W1-4

Month

If, following the year, is the character- and noWfollowing, then it will read the number following as the month of the year, starting from 1 being December. If after that number is another-, then it reads the following number as the day of the month, where 1 is the first day of the month.

Here's an example of a date made up of a year, a month and a day of the month. It is the same example as all above, representing January 1st, 2026: 

{
date: &2026-01-01

Time

If the first character is a T, then it will read the number following as the hour of the day, where 0 is 12am, 1 is 1am and 16 is 4pm of the same day. 24 is 12am of the next day.

If, following this number, which can be either 1 or 2 digits long, is a:, then it will read the following number as the minutes of the hour.

If, following this number, which can also be either 1 or 2 digits long, is another:, then it will read the following number as the seconds of the minute.

If, following this number, which can also be either 1 or 2 digits long, is a., then it will read the following number as the milliseconds of the second. It reads it as the decimals to the second, so .5 doesn't mean 5 milliseconds, it means half a second.

Here's some example usage of the time segment of a date (read comment): 

{
date1: &2026-07-01T12:30:05
date2: &T12:30:05.025

# These all represent 1:05am and 3.5 seconds
time1: &T01:05:03.5
time2: &T1:05:03.5
time3: &T01:5:3.50
time4: &T01:05:03.500
