Definition

In LXON, "color" is a way to group all color types into one. Color values encapsulate two distinct yet similar values types, as well as a subdivision in one:

  • !! val.color.srgb ! SRGB Color !#
  • !! val.color.linear ! Linear Color !:
    • !! val.color.linear.bit8 ! 8 Bit !=
    • !! val.color.linear.bit16 ! 16 Bit !/

Most languages and environments support both individual color types, however many don't. Many don't have an implementation at all, such as JavaScript. In those cases, both custom color objects are made. You can use then use those objects as a middleman for converting the LXON color value into a more fleshed out implementation from an external library (or other implementations).

Distinction

SRGB Colors and Linear Colors aren't just made up concepts by LXON. They're distinct and used in different situations.

This documentation won't go into depth on the difference between them, there are far better tutorials for that (although I personally couldn't find any good general use case ones). Feel free to search up a tutorial on the difference between both color spaces for your respective language/engine/environment.

If you never had to ask yourself which colorspace you're using, then you're most likely using SRGB. SRGB is used for web, photography, user interfaces, and countless other usecases and industries. If you're using Linear Color, you're most likely aware of it, although it is safe to make the verification.

As to the difference between 8 bit and 16 bit linear color, the difference is only a question of precision and limitation. If you're using linear color, use 8 bit unless you specifically need the extra color capacity. Same as with the question of SRGB to Linear, if you need 16 bit rather then 8 bit, you're likely aware of the difference already.
