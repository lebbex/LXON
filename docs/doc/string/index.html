Syntax

A string value is the more versatile alternative to !! values.text.full ! full string ! values. It requires a backtick` or caret^ at the start, as type declaration. Every following character is treated as the contents of the string, until the first instance either one of those characters. This means you start and end string characters with either of the same two characters.

Like with full strings, the string also ends at the end of the line, so trailing backticks aren't strictly required. They also support trailing whitespace such as tabs or space characters.

As a consequence of its increase in complexity, strings require escape sequences for the following characters:

  • Newline\n
  • Carriage Return\r
  • Backslash\\
  • Backtick\`
  • Caret\^

Chaining

Strings can be chained easily with the caret symbol^. When a caret is used to terminate a string, it will not only mark the end of the current value but also the start of a new string. This should only ever be done inside an !! containers.arr ! array !. It allows for string chaining, which is also supported by !! values.keybind ! keybinds !, !! values.enum ! enums ! and even !! values.binary ! binary ! values,

Important

Although tab representation is supported, it is standardized to replace them with\t. This allows for better visualization as well as a shortening of the visual length of the string. This does however come at the cost of an extra byte stored per tab, so this decision comes down to the developper.

Also, although trailing whitespace is supported, if there's no trailing backtick, it is standardized to replace the last character with a\sif it's a space, or a\tif it's a tab. Otherwise, it's easy to miss the trailing whitespace and be unclear as to whether it was intentionally done or not. For this reason, it is very possible for a friendly parser to trim leading whitespace, assuming it's a typing mistake.

Usage

Here's some example usage of string values as reference: 

{
key: `string`
ehh: `string

array:[ `str1` `str2`
	`str3
	`str4^str5`
	`str6^str7
