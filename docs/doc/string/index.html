Syntax

A string value is the more versatile alternative to !! values.text.full ! full string ! values. It requires a backtick` or caret^ at the start, as type declaration. Every following character is treated as the contents of the string, until the first instance either one of those characters. This means you start and end string characters with either of the same two characters.

Like with full strings, the string also ends at the end of the line, so trailing backticks aren't strictly required. Strings support !! other.escape ! escape characters !. They also support trailing whitespace such as tabs or space characters.

Chaining

Strings can be chained easily with the caret symbol^. When a caret is used to terminate a string, or even a !! values.keybind ! keybind !, a !! values.enum ! enum ! or a !! values.binary ! binary value, it will not only mark the end of the current value but also the start of a new string.

Important

Although newline termination is supported, it is highly reccomended to refrain from doing so, as it is standardized to always include a trailing backtick. Certain parsers might break without the usage of one. You can always use a full string in the event that you want to skip on a trailing backtick.

Also, although trailing whitespace is supported, if there's no trailing backtick, it is standardized to replace the last character with a\sif it's a space, or a\tif it's a tab. Otherwise, it's easy to miss the trailing whitespace and be unclear as to whether it was intentionally done or not. For this reason, it is very possible for a friendly parser to trim leading whitespace, assuming it's a typing mistake.
