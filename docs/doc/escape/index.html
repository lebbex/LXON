Justification

In many cases, data can't be represented as is because the parser is expecting a certain character. This is the reality in a lot of examples in programming.

To combat this, escape sequences were invented from the very surfacing of this problem. This problem is also faced in LXON, and the solution is the same implementation as everywhere.

Escape sequences are used when you want to include a character that would otherwise be impossible to include, whether it be as a typed key or as a value. For example, if you want to write a string with a newline, you would get this: 

{
text: "hello
world

However, this would simple parse into the following object (read !! other.comment ! comment !): 

{
text: "hello
world: _
# This ^ symbol means undefined

In order to parse "hello" and "world", seperated by a newline, you would need to use the escape character\n. 

{
text: "hello\nworld

This is incredibly useful in countless instances. Sroll to the bottom of the page to see all examples, including less obvious ones.

Syntax

A comment is very straightforward to write. Simply begin a new line with#and the rest of the line in full will be skipped by the parser.

Important

You cannot write inline comments, that means comments on the same line with data. Some friendly parsers will understand and keep the comment as a comment, however, to simplify the workload of most parsers, the standard is to only allow full-line comments. Writing inline comments would go against this standard and will likely break on many parsers, including the officially released ones.
