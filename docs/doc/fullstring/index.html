Syntax

A full string is the simplest way of representing a string in LXON. It requires only a double quote"at the start, as a type declaration. The entire rest of the line will be read as the string. Therefore, the only way to end a full string is with a newline or end of file.

Full strings support !! other.escape ! escape characters !. As a consequence of its simplicity however, the only characters needing escaping are newlines\n, carriage returns\r and backslashes\\. This makes them very desireable for string representation. Full strings also support trailing whitespace such as tabs or space characters.

Important

Although trailing whitespace is supported, it is standardized to replace the last character with a\sif it's a space, or a\tif it's a tab. Otherwise, it's easy to miss the trailing whitespace and be unclear as to whether it was intentionally done or not. For this reason, it is very possible for a friendly parser to trim leading whitespace, assuming it's a typing mistake.
