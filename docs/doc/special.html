Definition

A special value is a !! val.static ! static value ! that does not fit into any other value type. Both instances are values that, ironically, indicate a lack of value. Special values are one of the few types that don't have a !! keys.typed ! typed key ! alternative.

Syntax

Being a !! val.static ! static value !, special values don't acctually require any data to be stored. They only require the declaration character, that is either~for null or_for undefined. Once the parser reads this declaration character, it instantly completes the value.

Important

Many languages, environments and parsers do not support an undefined value type. If wishing to make use of "undefined", make sure your context supports the value, and if it does, make sure to understand it's exact implementation as it can differ depending on parser (custom "undefined" object, specific value, wildcard, etc.)
