Definition

Values represent the data itself, stored under either static or dynamic types.

Static Values

Static value are values that don't actually take in data. All they do is define a certain set value based off the value type definition symbol. Here is a cheat sheet list of all static values and their respective characters:

  • True$
  • False!
  • Null~
  • Undefined_
  • NaN+ -*Must be followed by a non-numerical character
  • Positive (+) Infinity% +%
  • Negative (-) Infinity-%

Dynamic Values

Dynamic values are values that take in data. They use a value type declaration symbol at the start, or are self defined in certain cases. Here is a cheat sheet list of all dynamic values and their respective declaration characters:

  • Full String"
  • String`
  • Char'
  • Number
  • Date&
  • Keybind@
  • SRGB Color#
  • Linear Color (8bit)=
  • Linear Color (16bit)/
  • Enum*
  • Binary?

Usage

To use a container, simply use its type declaration when defining a value, and the value will be the container itself. The container's content requires a further indentation level. To terminate the container, either subtract one indentation, or use its terminator symbol if and only if writing more data on that same line.
