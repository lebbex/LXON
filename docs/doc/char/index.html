Syntax

A char value is the single character alternative to !! val.text.full ! full string ! values. It requires a single quote'at the start, as type declaration. The following character is treated as the full contents of the value (or char if the language supports it).

Chars support !! other.escape ! escape characters !. Although this creates a value that appears to have two charactes, the\suffix gets skipped and is used for parsing the following character specially. As such, the value gets reduced to a single character and becomes a valid char value.

Important

The char value ends after the first character, so trailing single quotes don't acctually close the string, they announces the start of a new string character, similarly to the caret symbol^for !! values.text.string ! string ! values.

Although it's supported to type a litteral space or tab character following the single quote to set the value to such, it is standardized to use the !! other.escape ! escape character ! alternative of the whitespace (such as\sand\t respectively).

Usage

Here's some example usage of char values as reference: 

{
letterA: 'A
newline: '\n

char map:('
	A: "value
	B: "value
	\:: "value
